# Taxonomy

> Canonical naming conventions for the SaaS trading platform.
> All code, docs, UI copy, and database schemas must use these terms consistently.

---

## User & Auth

| Term | Definition | Usage |
|------|-----------|-------|
| **User** | A registered account | DB table, NextAuth subject |
| **Tier** | User's subscription level | Enum: `trial`, `standard`, `premium`, `suspended` |
| **Trial** | 7-day free period with Standard access, CC required upfront | First state after signup |
| **Standard** | $100/mo paid tier, 5 position checks/day during market hours | Tier enum value |
| **Premium** | $500/mo paid tier, hourly checks 24/7, strategy scans | Tier enum value |
| **Suspended** | Access revoked (payment failed, trial expired without payment) | Tier enum value |
| **Session** | Authenticated JWT context for a logged-in user | NextAuth session |

**Avoid:** "free tier", "basic", "pro", "plan" (use "tier"), "subscriber" (use "user"), "member".

## Chat & Messages

| Term | Definition | Usage |
|------|-----------|-------|
| **Chat** | A single conversation thread between user and agent | DB table, URL: `/chat/[id]` |
| **Message** | A single turn in a chat (user, assistant, or tool result) | DB table (`Message_v2`) |
| **Role** | Who authored the message | Enum: `user`, `assistant`, `tool` |
| **Part** | A segment of a message (text, file, tool call) | JSON array in Message.parts |
| **Attachment** | A file uploaded by the user within a message | JSON in Message.attachments |
| **Stream** | An active SSE response from the AI to the client | Resumable via Redis |
| **Artifact** | A structured document created by the AI (text, code, image, sheet) | Inherited from chatbot |

**Avoid:** "thread" (use "chat"), "response" for stored data (use "message"), "conversation" in code (use "chat").

## Trading

| Term | Definition | Usage |
|------|-----------|-------|
| **Trade** | A single executed order on Kalshi (immutable record) | DB table |
| **Position** | An open or closed holding resulting from a trade | DB table |
| **Order** | A request sent to Kalshi (may be resting or filled) | Kalshi API concept, not a DB table — we store the result as a Trade |
| **Recommendation** | A suggested trade generated by the agent or a scan | DB table |
| **Side** | Direction of a trade | Enum: `yes`, `no` |
| **Action** | Buy or sell | Enum: `buy`, `sell` |
| **Ticker** | Kalshi market identifier | e.g. `KXCPI-26MAR-T0.4` |
| **Event** | A Kalshi event containing one or more markets | e.g. `KXCPI-26MAR` |
| **Market** | A single binary contract within an event | Has a ticker, yes/no prices |
| **Entry Price** | Price in cents at which a position was opened | Field: `entryPriceCents` |
| **Current Price** | Latest bid price from Kalshi | Field: `currentPriceCents` |
| **Exit Price** | Price at which a position was closed | Field: `exitPriceCents` |
| **P&L** | Profit and loss in cents or dollars | Fields: `pnlCents`, `pnlDollars` |
| **Thesis** | The reasoning behind a trade or recommendation | Free text field |
| **Catalyst** | The external event driving a thesis | Free text field |
| **Edge** | Perceived mispricing (fair value minus market price) | Decimal |

**Avoid:** "bet" (use "trade"), "wager", "contract" alone (use "position" or "market"), "prediction" for a trade (use "recommendation").

### Position States

| State | Meaning |
|-------|---------|
| `open` | Active holding, not yet closed |
| `closed` | Exited via sell, settlement, or expiry |

### Recommendation States

| State | Meaning |
|-------|---------|
| `pending` | Generated, awaiting user action |
| `traded` | User executed the recommendation |
| `skipped` | User explicitly passed |
| `expired` | Market closed before user acted |

## Strategies

| Term | Definition | Usage |
|------|-----------|-------|
| **Strategy** | A saved, reusable trading playbook | DB table |
| **Strategy Config** | The JSON object defining a strategy's rules | Field: `Strategy.config` |
| **Template** | A built-in strategy that ships with the product | Strategy with `isTemplate: true` |
| **Custom Strategy** | A user-created strategy | Strategy with `isTemplate: false` |
| **Scan** | A single execution of a strategy's discovery pipeline | DB table |
| **Filters** | Keywords, event types, price ranges for market discovery | Config key: `filters` |
| **Entry Rules** | Conditions for opening a position (max price, min edge, min volume) | Config key: `entry_rules` |
| **Exit Rules** | Conditions for closing a position (target, stop loss, time decay) | Config key: `exit_rules` |
| **Sizing** | Position size parameters (max per trade, Kelly fraction, portfolio cap) | Config key: `sizing` |
| **Research Config** | Search query templates and source priority for the research phase | Config key: `research` |
| **Thesis Template** | Required documentation fields when logging a trade | Config key: `thesis_template` |

**Avoid:** "playbook" in code (use "strategy"), "rule set" (use "strategy config"), "bot" (use "strategy").

### Built-In Templates

| Slug | Name | Description |
|------|------|-------------|
| `oil` | Oil & Energy | Energy markets, commodity-driven CPI pass-through |
| `fat-tails` | Fat Tails | Low-probability events at extreme mispricings (≤5¢) |
| `volatility-swing` | Volatility Swing | Catalyst-driven price swings on political/economic events |
| `spread-arb` | Spread Arbitrage | Opposing market mispricings within the same event |

## Scheduled Operations

| Term | Definition | Usage |
|------|-----------|-------|
| **Dispatch** | The cron-triggered route that batches price checks and fans out scans | Route: `/api/jobs/dispatch` |
| **Position Check** | A lightweight, non-AI price update for all users' positions | Part of dispatch |
| **Strategy Scan** | An AI-powered analysis of a user's strategy against current markets | Fanned out via QStash |
| **Fan-Out** | Publishing individual jobs to QStash for parallel execution | Pattern name |
| **Snapshot** | An end-of-day portfolio and P&L recording | Route: `/api/jobs/daily-snapshot` |

**Avoid:** "job" alone without context (specify "cron job", "QStash job", or "scan"), "worker" (we don't have workers — use "cron route" or "scan route"), "task".

## Alerts

| Term | Definition | Usage |
|------|-----------|-------|
| **Alert** | A notification sent to a user about a position or opportunity | DB table |
| **Threshold Alert** | Triggered when a position's price crosses a target or stop loss | Alert type: `threshold` |
| **Opportunity Alert** | Triggered when a strategy scan finds an actionable candidate | Alert type: `opportunity` |
| **Fill Alert** | Triggered when a resting order fills | Alert type: `fill` |
| **Settlement Alert** | Triggered when a market resolves (settles) | Alert type: `settlement` |
| **Channel** | Delivery method for an alert | Enum: `email`, `in_app` |

**Avoid:** "notification" in code (use "alert"), "push notification" (we don't have mobile push).

## Billing

| Term | Definition | Usage |
|------|-----------|-------|
| **Subscription** | A recurring Stripe payment for a tier | Stripe object |
| **Checkout** | The Stripe-hosted payment collection flow | Stripe Checkout session |
| **Customer Portal** | Stripe self-service for payment method, invoices, cancellation | Stripe Customer Portal |
| **Webhook** | Stripe → our API callback for payment events | Route: `/api/webhooks/stripe` |
| **Grace Period** | Time after payment failure before suspension | Implementation detail |

**Avoid:** "plan" (use "tier"), "billing plan" (use "subscription"), "payment tier" (use "tier").

## AI & Tools

| Term | Definition | Usage |
|------|-----------|-------|
| **Agent** | The AI persona that users interact with | Product concept, not a code entity |
| **Tool** | A function the AI can invoke during a chat turn | AI SDK tools parameter |
| **Tool Call** | A single invocation of a tool by the AI | Logged as Message with `role: tool` |
| **Confirmation** | User approval required before executing a destructive tool | UI pattern for `createOrder`, `cancelOrder` |
| **System Prompt** | The initial instruction context sent to the LLM on each turn | Includes user state, strategy context |
| **AI Gateway** | Vercel's unified proxy to multiple LLM providers | Service: `@ai-sdk/gateway` |

### Tool Names (canonical)

| Tool | Category | Description |
|------|----------|-------------|
| `getMarkets` | Trading | Scan Kalshi events and markets |
| `getPositions` | Trading | User's balance, positions, resting orders |
| `createOrder` | Trading | Place a limit order (requires confirmation) |
| `cancelOrder` | Trading | Cancel a resting order (requires confirmation) |
| `webSearch` | Research | Multi-source web search |
| `xSearch` | Research | X/Twitter real-time sentiment search |
| `listStrategies` | Strategy | List user's saved strategies |
| `runStrategy` | Strategy | Execute a strategy's full scan pipeline |
| `saveStrategy` | Strategy | Create or update a strategy config |
| `getPortfolio` | Portfolio | Positions, P&L, performance stats |
| `getTradeHistory` | Portfolio | Past trades with outcomes |
| `createDocument` | Artifact | Create a text/code/image/sheet artifact (inherited) |
| `updateDocument` | Artifact | Modify an existing artifact (inherited) |
| `requestSuggestions` | Artifact | Generate writing suggestions (inherited) |

**Avoid:** "function" when referring to AI tools (use "tool"), "action" for AI tools (use "tool call"), "plugin".

## Database Naming Conventions

| Convention | Rule | Example |
|-----------|------|---------|
| Table names | PascalCase, singular | `User`, `Trade`, `Strategy` |
| Column names | camelCase | `entryPriceCents`, `createdAt`, `userId` |
| Money fields | Integer cents with `Cents` suffix | `priceCents`, `costCents`, `pnlCents` |
| Foreign keys | `{entity}Id` | `userId`, `chatId`, `strategyId` |
| Timestamps | `createdAt`, `updatedAt`, `closedAt` | Always `timestamp` type |
| Enums | Stored as `varchar`, values are lowercase | `tier: "standard"`, `status: "open"` |
| Boolean flags | `is` prefix | `isTemplate`, `isUpvoted`, `delivered` (exception: `delivered` for alerts) |

## API Route Naming

| Pattern | Convention | Example |
|---------|-----------|---------|
| REST resources | `/api/{resource}` | `/api/chat`, `/api/document` |
| Resource by ID | `/api/{resource}/[id]` | `/api/chat/[id]` |
| Cron jobs | `/api/jobs/{action}` | `/api/jobs/dispatch`, `/api/jobs/daily-snapshot` |
| QStash targets | `/api/jobs/scan-user/[userId]` | Per-user fan-out endpoint |
| Webhooks | `/api/webhooks/{provider}` | `/api/webhooks/stripe` |
| File uploads | `/api/files/upload` | Inherited from chatbot |

## UI Copy Conventions

| Context | Convention | Example |
|---------|-----------|---------|
| Tier names in UI | Title Case | "Standard", "Premium" |
| Trade actions | Lowercase verb | "buy", "sell" |
| Side labels | Lowercase | "yes", "no" |
| Money display | Dollar sign, 2 decimals | "$12.50", "$0.47" |
| Cents display | Number + ¢ | "12¢", "47¢" |
| Confidence | Title Case | "High", "Medium", "Low" |
| Dates | Relative when recent, absolute otherwise | "2 hours ago", "Feb 28, 2026" |
| Empty states | Action-oriented prompt | "No positions yet. Run a strategy to get started." |
